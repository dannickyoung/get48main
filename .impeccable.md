# get48 — Retainer Dashboard

## Design Context

### Users
- **Admin (single user — the owner):** runs a short-form video production studio on monthly retainers. Needs to add/remove clients, set each retainer's terms (start date, videos/month, price), record 50%/50% payments, log videos delivered, track overages, and — most importantly — see how many unused videos roll over each month under the rollover rules.
- **Clients (view-only):** each logs into a scoped account that shows only their own retainer: videos used vs. allotted this month, rolled-over balance and when it expires, overages, and payment status. They cannot edit anything.

### Brand Personality
Sharp, confident, operator-grade. Feels like a tool a busy studio owner actually trusts to be the source of truth on money and deliverables. Three words: **precise, calm, premium.** Not playful, not corporate-SaaS-generic.

### Aesthetic Direction
- **Theme:** dark. Studio owner checks it at odd hours; the data-density and "control room" feel suit dark. Reference: Supabase dashboard (deep near-black surfaces, restrained chrome, content-forward).
- **Accent:** lime green — used as the rare 10% accent (active states, key figures, primary actions, "on track" status), never as fill-everything.
- **Corners:** rounded (generous, consistent radius scale).
- **Anti-references:** generic admin templates, colored left-border cards, gradient text, glassmorphism, neon-on-black "AI" palette, identical card grids.

### Design Principles
1. **The rollover number is the hero.** The one thing this product exists to answer — "how many videos carry over and when do they expire" — should be the most legible thing on every relevant screen.
2. **Status at a glance, detail on demand.** Progressive disclosure: a client's health reads instantly; the math behind it expands when wanted.
3. **Lime is a scalpel, not a paintbrush.** Near-black tinted neutrals carry 90% of the UI; lime marks only what matters.
4. **Money is never ambiguous.** Payment state (deposit / balance / paid / overdue) is always unambiguous and never buried.
5. **No template smell.** Asymmetry, real typographic hierarchy, varied spacing rhythm — it should read as a bespoke studio tool, not a dashboard kit.
